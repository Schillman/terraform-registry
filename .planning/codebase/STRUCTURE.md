# Codebase Structure

**Analysis Date:** 2026-02-28

## Directory Layout

```
terraform-registry/
├── .github/
│   └── workflows/            # CI/CD pipeline definitions
├── .planning/
│   └── codebase/             # GSD analysis documents
├── modules/                  # Terraform modules (public API)
│   └── terraform-docker-container/
│       ├── tests/            # Module integration tests and examples
│       │   └── example/
│       ├── main.tf           # Resource definitions
│       ├── variables.tf       # Input variable declarations
│       ├── terraform.tf       # Provider version requirements
│       └── README.md          # Module documentation
├── .gitignore                # Git exclusion patterns (Terraform standard)
├── .markdownlint.yml         # Markdown linting configuration
└── README.md                 # Repository documentation
```

## Directory Purposes

**modules/:**
- Purpose: Public API - contains all reusable Terraform modules in the registry
- Contains: Provider-specific modules following naming pattern `terraform-{provider}-{module_name}`
- Key files: Each module has main.tf, variables.tf, terraform.tf, README.md

**modules/terraform-docker-container/:**
- Purpose: Single module for Docker container provisioning
- Contains: Complete module implementation with configuration, resources, and examples
- Key files: `main.tf`, `variables.tf`, `terraform.tf`, `README.md`

**modules/terraform-docker-container/tests/:**
- Purpose: Holds test configurations and examples demonstrating module usage
- Contains: Working example that can be executed to test the module
- Key files: `tests/example/main.tf` shows real-world usage pattern

**.github/workflows/:**
- Purpose: Defines automated CI/CD checks on push and pull requests
- Contains: Linting, formatting validation, and security checks
- Key files: `lint.yaml` runs Terraform format checks and markdown linting

**.planning/codebase/:**
- Purpose: Generated GSD analysis documents (non-committed planning artifacts)
- Contains: ARCHITECTURE.md, STRUCTURE.md, and other analysis docs
- Generated: Yes

## Key File Locations

**Entry Points:**
- `modules/terraform-docker-container/main.tf`: Primary resource definitions; terraform processes this when module is invoked
- `.github/workflows/lint.yaml`: CI entry point; runs on every push/PR to main branch

**Configuration:**
- `modules/terraform-docker-container/variables.tf`: Declares all valid inputs and constraints
- `modules/terraform-docker-container/terraform.tf`: Specifies required provider (Docker 3.0.2)
- `.markdownlint.yml`: Markdown documentation linting rules (line length 180, HTML elements)

**Core Logic:**
- `modules/terraform-docker-container/main.tf`: Implements three resource types:
  - `docker_image.main` - Pulls specified image
  - `docker_volume.volumes` - Creates volumes with optional labels
  - `docker_container.main` - Provisions container with dynamic healthcheck, devices, ports, volumes

**Testing:**
- `modules/terraform-docker-container/tests/example/main.tf`: Example module invocation with Ubuntu container, bridge network, volume mount

**Documentation:**
- `README.md`: Root readme describing module registry purpose and how to use/add modules
- `modules/terraform-docker-container/README.md`: Module-specific documentation (autogenerated with terraform-docs)

## Naming Conventions

**Files:**
- `main.tf`: Primary resource definitions (Terraform standard)
- `variables.tf`: Input variable declarations (Terraform standard)
- `terraform.tf`: Provider requirements (Terraform standard)
- `outputs.tf`: Output value declarations (not used in current module)
- `README.md`: Module documentation

**Directories:**
- `terraform-{provider}-{module_name}`: Module naming pattern (e.g., `terraform-docker-container`)
- `tests/example/`: Example configurations location
- `.planning/codebase/`: Analysis documents directory

**Variables:**
- snake_case for variable names: `docker_image_name`, `environment_variables`, `network_mode`
- Prefixed by resource type when relevant: `docker_image_name` (Docker image), `network_mode` (network)

**Resources:**
- Lowercase with underscores: `docker_image.main`, `docker_volume.volumes`, `docker_container.main`
- Logical names: `.main` for primary resource, `.volumes` for collection

## Where to Add New Code

**New Terraform Module:**
- Primary code: Create `modules/terraform-{provider}-{module_name}/` directory
- Required files: `main.tf`, `variables.tf`, `terraform.tf`, `README.md`
- Tests: Add `tests/example/main.tf` with working example
- Documentation: Auto-generate with terraform-docs in README using markers `<!-- BEGIN_TF_DOCS -->` and `<!-- END_TF_DOCS -->`

**New Resources to Existing Module:**
- Implementation: Add resource blocks to `modules/terraform-docker-container/main.tf`
- Inputs: Add corresponding variables to `modules/terraform-docker-container/variables.tf` with type constraints
- Documentation: Update `modules/terraform-docker-container/README.md` (will be reflected in autogenerated section)
- Example: Update `tests/example/main.tf` to demonstrate new configuration options

**New Provider Module:**
- Copy `modules/terraform-docker-container/` structure as template
- Update `terraform.tf` with new provider name and version
- Update `main.tf` with provider-specific resources
- Update `variables.tf` with new provider's input options
- Create `tests/example/main.tf` with working configuration
- Follow naming: `terraform-{provider}-{module_name}`

**Utilities:**
- Terraform modules are self-contained; shared code pattern not used currently
- Reusable patterns: Use module composition (one module calling another) for code reuse

## Special Directories

**node_modules / .terraform:**
- Purpose: Dependency cache (Terraform modules, provider binaries)
- Generated: Yes (created by `terraform init`)
- Committed: No (listed in .gitignore as `**/.terraform/*`)

**tfstate Files:**
- Purpose: Terraform state tracking
- Generated: Yes (created by `terraform apply`)
- Committed: No (listed in .gitignore as `*.tfstate`, `*.tfstate.*`)

**.tfvars Files:**
- Purpose: Input variable values (often contains sensitive data)
- Generated: User-created
- Committed: No (listed in .gitignore as `*.tfvars`, `*.tfvars.json`)

**.terraform.lock.hcl:**
- Purpose: Locks dependency versions for reproducibility
- Generated: Yes (created by `terraform init`)
- Committed: Yes (present in `tests/example/`)
