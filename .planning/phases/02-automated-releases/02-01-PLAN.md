<task type="auto">
  <name>Task 1: Create release.yaml — terraform-module-releaser workflow</name>
  <files>.github/workflows/release.yaml</files>
  <action>
Create `.github/workflows/release.yaml` with the following exact content:

```yaml
---
name: Terraform Module Releaser

on:
  pull_request:
    types: [opened, reopened, synchronize, closed]
    branches:
      - main

permissions:
  contents: write
  pull-requests: write

jobs:
  release:
    name: Release Terraform Modules
    runs-on: ubuntu-latest
    steps:
      - name: Checkout
        uses: actions/checkout@v4
        with:
          fetch-depth: 0

      - name: Terraform Module Releaser
        uses: techpivot/terraform-module-releaser@v2
```

Critical requirements (do NOT deviate):
- `permissions: contents: write, pull-requests: write` MUST be at the workflow level (not job level). Without this, GITHUB_TOKEN defaults to read-only and the action exits green without creating any tags.
- `fetch-depth: 0` MUST be present. The action uses full git history for tag detection. Default shallow checkout breaks it.
- Trigger MUST be `pull_request` with all four types `[opened, reopened, synchronize, closed]`. The action creates the actual tag when the PR is closed/merged. Using `push` instead will not trigger the release.
- Use `actions/checkout@v4` to match the existing lint.yaml pattern (not @v6).
- No additional inputs are needed — the action auto-detects modules by scanning for directories containing .tf files recursively from the workspace root. The `modules/docker/container/` directory will be detected correctly.
  </action>
  <verify>

<task type="auto">
  <name>Task 2: Create pr-title.yaml — PR title conventional commits validation</name>
  <files>.github/workflows/pr-title.yaml</files>
  <action>
Create `.github/workflows/pr-title.yaml` with the following exact content:

```yaml
---
name: Lint PR Title

on:
  pull_request:
    types: [opened, reopened, edited, synchronize]
    branches:
      - main

permissions:
  pull-requests: write

jobs:
  validate-pr-title:
    name: Validate PR Title
    runs-on: ubuntu-latest
    steps:
      - uses: amannn/action-semantic-pull-request@v5
        env:
          GITHUB_TOKEN: ${{ secrets.GITHUB_TOKEN }}
        with:
          types: |
            feat
            fix
            docs
            chore
            refactor
            test
            ci
```

Notes:
- Use `pull_request` (not `pull_request_target`). This repo has no fork-based PR workflow.
- `pull-requests: write` permission allows the action to post a comment explaining WHY the title was rejected. Without it, the check fails but the contributor has no feedback.
- The `types` list matches SKILL.md conventional commit types exactly: feat, fix, docs, chore, refactor, test, ci.
- The `edited` type is included so re-editing the PR title re-triggers validation.
- `amannn/action-semantic-pull-request@v5` automatically handles the `!` suffix for breaking changes (feat!, fix!) — no additional configuration needed.
  </action>
  <verify>
